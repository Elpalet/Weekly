# DOCX 格式规范

## 样本依据

规范来自本地历史周报样本中最近一年 42 份 DOCX（2025-05-19 至 2026-04-19）：

- 42/42 均含 `工作内容：`、`问题：`、`下周计划：`，且顺序一致。
- 42/42 使用 A4 纵向，页边距为上/下 `2.54 cm`、左/右 `3.17 cm`。
- 文档默认字号为 `10.5 pt`；主体以 Word `List Paragraph` 样式为主，解释性长段落可使用 `Normal`。
- 未发现页眉或页脚。
- 39/42 含图片，共 196 张；仅 2/42 使用表格。图片是常规证据载体，表格不是默认布局。

## 新文档版式

- 页面：A4 纵向，宽 `21.0 cm`，高 `29.7 cm`。
- 页边距：上 `2.54 cm`，下 `2.54 cm`，左 `3.17 cm`，右 `3.17 cm`；正文可用宽度约 `14.66 cm`。
- 字号：正文 `10.5 pt`，沿用简洁的 Word 默认中文正文观感；不另设大号标题。
- 开篇：首行直接为 `工作内容：`，不加入姓名、日期大标题、封面、目录或摘要。
- 段落：主要使用 `List Paragraph` 组织工作条目、问题与计划；较长的方法说明或图注可使用 `Normal`。
- 标题：三个固定分区与正文保持同一朴素体系，不使用彩色、底纹、复杂编号或装饰线。
- 页眉页脚：保持为空，除非用户明确要求添加。

## 图表处理

- 用户提供结果图、流程图或截图时，把图放在对应叙述之后，宽度不超过正文可用宽度。
- 保留图前后的必要解释，使图片对应到具体项目或判断。
- 只有在重复字段需要比较时才使用表格；普通任务说明、问题和计划不要改成表格。

## 输出与校验

1. 文件名使用 `周报YYYYMMDD-YYYYMMDD.docx`，日期对应报告周期。
2. 使用文档/DOCX 技能生成 Word 文件，明确设置页面尺寸、页边距和字号，不依赖通用模板默认值。
3. 若环境可渲染 DOCX，输出前渲染全部页面并检查图片、换页、缩进和中文字体显示。
4. 若无法渲染，至少检查 DOCX 结构、三个固定分区、页面参数、图片尺寸与文件可打开性，并说明未完成视觉渲染验证。
5. 历史周报只作为版式与语言参考，不修改、不覆盖，也不把既有工作内容带入新周报。
